# User Flow 定義規則

## 目標

根據需求列出所有使用者會遇到的情境，用 BDD（Given-When-Then）格式描述。專注在「使用者怎麼用這個功能」，不是技術邊界測試。

## 規則

- 用白話文寫，不需要任何程式碼或技術術語
- 每個情境都是從使用者角度出發：使用者做了什麼、期望看到什麼
- 不要寫邊界測試（密碼長度、欄位格式驗證等），那是技術層的事
- 專注在使用者實際會遇到的操作流程

## 輸出位置

`tasks/userflow.md`

## 範例

需求：製作註冊功能

```markdown
## 使用者註冊

### 情境：使用者成功註冊
- Given 使用者在註冊頁面
- When 填寫完資料並送出
- Then 註冊成功並導向首頁

### 情境：使用者用已存在的信箱註冊
- Given 使用者在註冊頁面
- When 填寫了一個已被使用的信箱並送出
- Then 顯示信箱已被註冊的提示

### 情境：使用者從註冊頁切換到登入頁
- Given 使用者在註冊頁面
- When 點擊「已有帳號？登入」
- Then 導向登入頁面
```
