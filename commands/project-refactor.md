# Flow

- 遵循此 Flow 順序一步一步往下做

1. 用以下目標呼叫多個 Explore Agent 起來搜尋，並根據難易度使用不同模型
  - 找出濫用 try-catch 的程式碼，try-catch 應該只會用在第三方服務 (I/O 等不是第三方)，其餘應使用 early return 來提前回傳錯誤 -> haiku
  - 找出複雜度過高的程式碼，用 if/switch 等判斷是做判斷，一個就是 1 點，如果是 nest 結構則為 2 點，如果超出 6 點則是為複雜度偏高需要重構 -> sonnet
  - 找出寫了無意義註解的地方，註解應該要寫的是這段程式碼**為什麼這樣做**，而不是這段程式碼在做什麼，同時，不該在複雜度不高 (6點以下) 的情況下寫註解 -> haiku
  - 找出沒有在使用的程式碼、Function -> haiku
  - 找出重複度太多 (3 次) 的程式碼，並抽成共用或者其他方式等，簡化邏輯 -> sonnet
  - 對於程式碼的可讀性，我希望以 AI 比較好理解的方式撰寫，而非人類好理解的方式 -> sonnet
