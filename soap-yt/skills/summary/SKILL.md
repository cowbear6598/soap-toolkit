# 工作流程

讀取 SRT 字幕檔，理解完整內容後，以繁體中文產出結構化摘要。輸出為 Slack mrkdwn 格式的純文字檔（`summary.txt`），放在影片所在資料夾。

## 步驟

1. 讀取完整字幕內容（包含時間戳記）
2. 理解全篇脈絡後，將內容翻譯 / 整理為繁體中文
3. 以書的結構組織：目錄 → 各章節大綱與內容
4. 根據 SRT 時間戳記，標註每個章節對應的影片時間範圍

## 輸出格式

使用 Slack mrkdwn 語法：

```
:movie_camera: *《影片標題》重點整理*

:bookmark_tabs: *目錄*
1. 章節名稱（00:00 - 05:30）
2. 章節名稱（05:30 - 12:15）
3. 章節名稱（12:15 - 18:40）

---

*1. 章節名稱*（00:00 - 05:30）
> 該章節的核心觀點與重點內容，以 2-3 句話說明。

*2. 章節名稱*（05:30 - 12:15）
> 該章節的核心觀點與重點內容，以 2-3 句話說明。

*3. 章節名稱*（12:15 - 18:40）
> 該章節的核心觀點與重點內容，以 2-3 句話說明。
```

## 注意事項

- 所有內容皆為繁體中文（不論原始語言）
- 先通盤理解再組織，不是逐段翻譯
- 章節劃分依內容主題，不是依字幕時間切割
- 時間範圍從 SRT 時間戳記推算：章節起始時間 = 該主題第一句字幕的開始時間，結束時間 = 該主題最後一句字幕的結束時間
- 時間格式統一為 `MM:SS`，超過一小時則用 `H:MM:SS`
- 使用 `*bold*` 而非 markdown 的 `**bold**` 或 `#` 標題
- 使用 `>` 引言區塊放內容
- 整體長度控制在適合 Slack 訊息閱讀的範圍內
