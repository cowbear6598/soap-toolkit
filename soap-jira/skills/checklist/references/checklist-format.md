# 驗收清單格式規範

## 格式規則

- 使用大寫英文字母加句號作為分組標題前綴（A. B. C.）
- 每個測試項目以 `[]` 開頭，表示未勾選的 checkbox
- 分組之間空一行
- 不加額外的 Markdown 標記（不用 #、不用 -、不用粗體）

## 範例

```
A.帳號管理
[]可以建立新帳號，建立後列表顯示該帳號
[]可以編輯帳號名稱，儲存後顯示更新後的名稱
[]可以停用帳號，停用後該帳號無法登入
[]建立帳號時未填必填欄位，顯示錯誤提示

B.權限設定
[]可以為帳號指派角色，指派後該帳號具備對應權限
[]移除角色後，該帳號無法存取受限功能
[]確認無權限的使用者看不到管理選單

C.通知機制
[]帳號建立成功後，該使用者收到啟用通知信
[]密碼重設後，該使用者收到密碼變更通知
[]通知信內容包含正確的帳號名稱與操作時間
```

## 待確認項目

當需求描述不明確、沒有定義具體行為時，使用以下格式標記：

```
[]⚠️ 待確認：批次贈送部分失敗時的處理機制（全部回滾？還是成功的照常派發？）
```

## 注意事項

- 測試項目描述操作和預期結果，用逗號或句號連接
- 每條控制在一行內，不換行
- 分組數量依實際功能模組決定，不硬性限制
- 待確認項目用 `⚠️ 待確認：` 前綴標記，讓審核員一眼看出需要補充的地方
