# 測試覆蓋 Review 規則

## 檢查範圍

- 這次更動到的檔案與其實作內容
- 檢查測試是否有覆蓋或偏離更新後的實作內容
- 檢查是否有測試使用 skip 或者直接讓測試無腦通過的方式，這是嚴重錯誤

## 核心原則

- 注重 User Flow 而不是無意義的邊界測試
- 思考是否有更多測試被遺漏

## 輸出格式

```markdown
## 測試覆蓋 Review 結果

### ❌ 需要補測試

#### register.ts
- 註冊成功
- 註冊失敗
- email 重複要回傳錯誤訊息

#### orderService.ts
- 建立訂單成功
- 庫存不足時建立訂單失敗

### ⚠️ 測試品質問題
| 檔案 | 問題 | 說明 |
|------|------|------|
| test/auth.test.ts | 使用 skip | 不應跳過測試 |

```

若沒有任何問題，回報單行「無問題」即可，不列檔案清單或表格。
